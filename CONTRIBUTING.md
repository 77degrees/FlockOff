# How to contribute
First off, please do! :)

## Find an issue?
If you find an issue, check to see if there's already an issue created for what you found.  If there is, please add whatever detail that may help.  If there isn't, please create a new one.

## Development
To start, fork the project and create a new branch to work in.  When you're ready, create a PR in this repo.  Wondering how to start with that?  There are a ton of resources out there for this step that explain it all better than I could.

Take a look at the `DEVELOPING.md` doc for some help getting the environment set up and a few notes.

### Code reviews
Nobody's perfect, especially me!  Code reviews are a great way to contribute to any open source project.

### Hardware
The current PCB and component selection are based around what I had in my home stock and what I could find on the cheap.  Nothing is really set in stone; for example, any GPS that works with NMEA sentences should work fine.

The current 3D case design isn't very well thought out and could surely be improved.

### Coding and development
Like c++?  Python3 for the local management?  Have a great idea for updated/new feature?  Let's goooooo!  Clone it or fork it; either way, get the code and start hacking.

### Documentation
The original developer (ahem, cough, cough... me!) didn't necessarily do a good job with documentation/commenting.  Any work there would be appreciated.

## But some notes and rules...

If this is confusing or offensive, please find another project to work with.  **This is non-negotiable**. Sorry, my project, my rules.

+ Rule number two - be nice.  I know rule number one does not sound nice; consider that one tough love.

+ Rule number three - this project is for fun.  Have fun, learn, and grow!
