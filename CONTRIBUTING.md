# How to contribute
First off, please do! :)

## Find an issue?
First, check to see if there's already an issue created for what you found.  If there is, please add whatever detail that may help.  If there isn't, please create a new one.

## Development
First off, fork the project and create a new branch to work in.  When you're ready, create a PR in this repo.  Wondering how to start with that?  There are a ton of resources out there for this step.

Take a look at the `DEVELOPING.md` doc for some help getting the environment set up and a few notes.

### Code reviews
Nobody's perfect, especially me!  Code reviews are a great way to contribute to any open source project.

### Coding and development
Like c++?  Python3 for the local management?  Have a great idea for updated/new feature?  Let's goooooo!

### Documentation
The original developer (ahem, cough, cough... me!) didn't necessarily do a good job with documentation/commenting.  Any work there would be appreciated.

## But some notes and rules...
+ Number one rule - any PR with content derived from an LLM/Agent/any-fucking-AI-thing will be immediately closed.  There will be none of that slop in this project.  If this is confusing or offensive, please find another project to work with.  **This is non-negotiable**. Sorry, my project, my rules.

+ Rule number two - be nice.  I know rule number one does not sound nice; consider that one tough love.

+ Rule number three - this project is for fun.  Have fun, learn, and grow!
